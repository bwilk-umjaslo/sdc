# Lista kontrolna archiwizacji

## Cel

Celem listy jest potwierdzenie, że zasób cyfrowy może zostać zarchiwizowany bez utraty treści, metadanych, kontekstu, integralności, dostępności i odpowiedzialności.

## Zakres

Lista dotyczy dokumentów elektronicznych, treści WWW i BIP, załączników, multimediów, danych z systemów, paczek archiwalnych i zasobów historycznych. Stosuje się ją przed nadaniem statusu archiwalnego, przekazaniem, migracją archiwalną albo decyzją o dalszym przechowywaniu.

## Zasady

Odpowiedzi należy oznaczać jako: TAK, NIE albo NIE DOTYCZY. Odpowiedź NIE przy punkcie blokującym zatrzymuje archiwizację do czasu uzupełnienia braków albo udokumentowania odstępstwa zatwierdzonego przez właściwą rolę.

## Szczegóły

| Obszar | Pytanie | TAK | NIE | NIE DOTYCZY | Punkt blokujący |
| --- | --- | --- | --- | --- | --- |
| Identyfikacja | Czy zasób ma jednoznaczną nazwę albo identyfikator? |  |  |  | Tak |
| Identyfikacja | Czy wskazano lokalizację, URL albo system źródłowy? |  |  |  | Tak |
| Metadane | Czy uzupełniono właściciela, daty, typ, format, status i źródło? |  |  |  | Tak |
| Metadane | Czy zapisano powiązanie ze sprawą, projektem albo procesem? |  |  |  | Tak, jeżeli dotyczy dokumentacji |
| JRWA | Czy ustalono klasę JRWA albo przyczynę braku klasy? |  |  |  | Tak, jeżeli zasób jest dokumentacją |
| JRWA | Czy ustalono kategorię archiwalną i okres przechowywania? |  |  |  | Tak, jeżeli zasób jest dokumentacją |
| Status | Czy obecny status jest zgodny z [Modelem statusów zasobów](./07-model-statusow-zasobow.md)? |  |  |  | Tak |
| Status | Czy wskazano status po archiwizacji? |  |  |  | Tak |
| Dostępność | Czy oceniono dostępność cyfrową zasobu? |  |  |  | Tak, jeżeli zasób jest albo będzie udostępniany |
| Dostępność | Czy wskazano dostęp alternatywny, jeżeli zasób jest niedostępny? |  |  |  | Tak, jeżeli dotyczy |
| Integralność | Czy zabezpieczono oryginał albo kopię kontrolną? |  |  |  | Tak |
| Integralność | Czy wykonano sumę kontrolną, blokadę edycji albo inny środek integralności? |  |  |  | Tak, jeżeli zasób ma wartość dowodową |
| Format | Czy format umożliwia odczyt w czasie albo zaplanowano konwersję? |  |  |  | Tak |
| Format | Czy zachowano oryginał przy konwersji? |  |  |  | Tak, jeżeli wykonano konwersję |
| Decyzja | Czy decyzję podjął właściwy właściciel albo kierownictwo? |  |  |  | Tak |
| Decyzja | Czy uwzględniono archiwum zakładowe, IOD, IT albo koordynatora dostępności, jeżeli było to wymagane? |  |  |  | Tak, jeżeli dotyczy |
| Dokumentowanie | Czy decyzję zapisano w formularzu albo rejestrze? |  |  |  | Tak |
| Dokumentowanie | Czy wskazano sposób dalszego dostępu do zasobu? |  |  |  | Tak |

Punkty blokujące obejmują brak identyfikacji zasobu, brak właściciela, brak metadanych minimalnych, brak kwalifikacji archiwalnej dla dokumentacji, brak decyzji, brak zabezpieczenia integralności oraz brak sposobu dalszego dostępu.

## Ryzyka

Pominięcie listy może prowadzić do archiwizacji pozornej, w której zasób jest tylko przeniesiony do folderu, ale nie ma statusu, kontekstu, kategorii, metadanych ani możliwości odczytu.
