# Formularz decyzji o wycofaniu

## Cel

Celem formularza jest zapewnienie jednolitego zapisu decyzji o wycofaniu treści z publikacji, wycofaniu zasobu z użycia albo wyłączeniu systemu. Formularz ma potwierdzić, że przed wycofaniem oceniono wpływ na dane, użytkowników, archiwizację, migrację i komunikację.

## Zakres

Formularz stosuje się do wycofania stron WWW, wpisów BIP, plików, multimediów, formularzy, aplikacji, CMS, systemów dziedzinowych, usług zewnętrznych i innych zasobów cyfrowych.

## Zasady

Formularz należy wypełnić przed wykonaniem czynności technicznej. Wycofanie z publikacji, usunięcie z widoku użytkownika, zamknięcie aplikacji albo wyłączenie systemu wymaga decyzji, a nie tylko zgłoszenia technicznego.

### Opis pól

| Pole | Sposób wypełnienia |
| --- | --- |
| Przedmiot | Treść, zasób, zbiór danych, aplikacja, system albo usługa objęta wycofaniem. |
| Powód | Nieaktualność, koniec projektu, zmiana systemu, ryzyko prawne, koniec wsparcia, zastąpienie nową wersją albo inna przesłanka. |
| Zakres danych | Dane, dokumenty, metadane, załączniki, logi, multimedia, konta i relacje objęte decyzją. |
| Wpływ | Wpływ na użytkowników, sprawy, informację publiczną, BIP, dostępność, dane osobowe i ciągłość działania. |
| Migracja | Zakres migracji, system docelowy, format eksportu, test odtworzenia albo informacja, że migracja nie dotyczy. |
| Archiwizacja | Sposób zachowania zasobu, kategoria, status, miejsce przechowywania, dostęp na wniosek albo przekazanie. |
| Komunikacja | Komunikat dla użytkowników, przekierowania, kontakt, kanał zastępczy, termin i odpowiedzialny. |
| Decyzja | Wycofać, wycofać po spełnieniu warunków, utrzymać, poprawić, migrować, archiwizować, przekazać albo wstrzymać do czasu uzupełnienia braków. |

## Ryzyka

Brak formularza zwiększa ryzyko wycofania bez podstawy, utraty danych, braku komunikacji, naruszenia dostępu do informacji publicznej, pominięcia IOD, pominięcia archiwum zakładowego i wyłączenia systemu bez testu odtworzenia.
