# Słownik pojęć

## Cel

Celem słownika jest ujednolicenie pojęć stosowanych w zaleceniu. Definicje mają charakter operacyjny i służą stosowaniu procedury w podmiocie publicznym.

## Pojęcia ogólne

| Pojęcie | Definicja |
| --- | --- |
| Archiwum państwowe | Właściwa jednostka państwowej sieci archiwalnej, z którą podmiot uzgadnia sprawy materiałów archiwalnych i brakowania w zakresie wymaganym przepisami. |
| Archiwum zakładowe | Komórka albo funkcja w podmiocie odpowiedzialna za przejmowanie, ewidencjonowanie, przechowywanie i udostępnianie dokumentacji zgodnie z instrukcją archiwalną. |
| Autentyczność | Możliwość wykazania pochodzenia, autorstwa, kontekstu i wiarygodności zasobu. |
| Dane systemowe | Dane przechowywane w systemie, które opisują działanie, konfigurację, relacje, logi, uprawnienia albo historię zasobu. |
| Dane osobowe | Dane dotyczące osoby fizycznej, które wymagają oceny przed publikacją, dalszym przechowywaniem, migracją, ograniczeniem dostępu albo anonimizacją. |
| Dokument elektroniczny | Dokument utrwalony w postaci cyfrowej, który może być częścią akt sprawy, publikacji, systemu albo paczki archiwalnej. |
| Dokument sprawy | Dokument powiązany z prowadzoną sprawą, który powinien mieć miejsce w aktach sprawy niezależnie od tego, czy został opublikowany w WWW albo BIP. |
| Informacja publiczna | Informacja dotycząca spraw publicznych, którą podmiot ocenia pod kątem publikacji, dalszego publicznego dostępu albo udostępnienia na wniosek. |
| Integralność | Właściwość zasobu potwierdzająca, że jego treść, dane, metadane albo relacje nie zostały zmienione w sposób nieuprawniony. |
| Kategoria A | Kategoria materiałów archiwalnych przeznaczonych do trwałego przechowywania. |
| Kategoria B | Kategoria dokumentacji niearchiwalnej przechowywanej przez określony czas, po którym może podlegać brakowaniu zgodnie z procedurą. |
| Kategoria archiwalna | Oznaczenie wskazujące, czy dokumentacja ma być przechowywana trwale, czy przez określony czas przed dalszą decyzją. |
| Kopia zapasowa | Kopia danych albo systemu służąca do odtworzenia zasobu po błędzie, awarii, incydencie albo nieudanej migracji. |
| Log | Zapis zdarzeń w systemie, który pomaga ustalić czynności wykonane na zasobie, użytkownika, czas, zakres zmiany i skutek. |
| Metadane | Dane opisujące zasób, w szczególności identyfikator, nazwę, właściciela, daty, status, format, źródło, kategorię, lokalizację i powiązanie ze sprawą. |
| Repozytorium | Miejsce przechowywania zasobów, danych, plików albo paczek archiwalnych wraz z zasadami dostępu i utrzymania. |
| System dziedzinowy | System obsługujący określony obszar działania podmiotu, w którym mogą powstawać dane, dokumenty, załączniki i metadane wymagające archiwizacji. |
| Status zasobu | Oznaczenie etapu lub sposobu traktowania zasobu, które wskazuje, czy zasób jest aktywny, archiwalny, wycofany, migrowany, do przeglądu albo wymaga uzupełnienia. |
| Treść | Informacja przeznaczona do odczytu przez użytkownika, publikowana albo przechowywana jako strona, wpis, dokument, załącznik, multimedia lub komunikat. |
| Treść archiwalna | Treść, która nie pełni już funkcji bieżącej, ale została zachowana i może być dostępna publicznie albo na wniosek ze względu na wartość informacyjną, dokumentacyjną albo historyczną. |
| Zasób cyfrowy | Dokument, treść, dane, plik, multimedia, rekord, formularz, wpis, eksport, log albo inny obiekt przechowywany, przetwarzany lub udostępniany w postaci cyfrowej. |
| Zasób historyczny | Zasób zachowany ze względu na znaczenie dla historii sprawy, projektu, publikacji, działania podmiotu, informacji publicznej albo ciągłości wiedzy organizacyjnej. |

## Procesy i działania

| Pojęcie | Definicja |
| --- | --- |
| Archiwizacja cyfrowa | Zespół czynności służących zachowaniu zasobu cyfrowego z treścią, metadanymi, kontekstem, integralnością i możliwością odczytu w czasie. |
| Archiwizacja formalna | Postępowanie z dokumentacją zgodnie z instrukcją kancelaryjną, JRWA, instrukcją archiwalną i decyzjami właściwego archiwum. |
| Archiwizacja operacyjna | Zachowanie zasobu w sposób pozwalający podmiotowi nadal go odszukać, odczytać, wyjaśnić jego kontekst i obsłużyć dalszy dostęp. |
| Anonimizacja | Trwałe usunięcie albo przekształcenie danych osobowych w taki sposób, aby zasób mógł być dalej używany bez identyfikacji osób. |
| Brakowanie | Formalne niszczenie albo usuwanie dokumentacji kategorii B po upływie okresu przechowywania, po kwalifikacji i po spełnieniu wymaganej procedury. |
| Decyzja tymczasowa | Decyzja zabezpieczająca zasób do czasu ustalenia właściciela, kategorii archiwalnej, podstawy dostępu albo sposobu odtworzenia. |
| Eksport danych | Przygotowanie danych, plików, metadanych i relacji do przekazania poza system źródłowy w uzgodnionym formacie. |
| Inwentaryzacja | Ustalenie, jakie zasoby, dane, dokumenty, systemy, relacje i zależności muszą zostać objęte decyzją, migracją albo archiwizacją. |
| Kwalifikacja archiwalna | Ustalenie właściwej klasy JRWA, kategorii archiwalnej i okresu przechowywania dla dokumentacji albo zasobu powiązanego ze sprawą. |
| Migracja danych | Przeniesienie danych, dokumentów, metadanych i relacji do innego systemu, formatu albo repozytorium z potwierdzeniem możliwości odtworzenia. |
| Odtworzenie zasobu | Przywrócenie zasobu, jego treści, metadanych albo kontekstu z kopii, eksportu, systemu źródłowego, akt sprawy lub innych zachowanych materiałów. |
| Paczka archiwalna | Uporządkowany zestaw treści, danych, metadanych, dokumentacji i potwierdzeń integralności przygotowany do przechowywania albo przekazania. |
| Przegląd zasobu | Ocena aktualności, statusu, właściciela, kategorii archiwalnej, dostępności, ryzyk i dalszego sposobu postępowania z zasobem. |
| Protokół braków | Dokument opisujący, których elementów zasobu, systemu, metadanych, logów albo relacji nie udało się odtworzyć lub przekazać. |
| Przekazanie do archiwum zakładowego | Czynność organizacyjna polegająca na przekazaniu dokumentacji do archiwum zakładowego wraz z wymaganą ewidencją i metadanymi. |
| Retencja | Okres i sposób przechowywania dokumentacji, danych albo zasobów wynikający z JRWA, przepisów, decyzji organizacyjnej albo potrzeby dowodowej. |
| Test odtworzenia | Sprawdzenie, czy zasób, eksport, kopia albo paczka archiwalna mogą zostać odczytane wraz z metadanymi i podstawowym kontekstem. |
| Uzupełnienie metadanych | Dodanie albo poprawienie danych opisowych potrzebnych do identyfikacji, wyszukania, oceny, udostępnienia lub archiwizacji zasobu. |
| Wariant minimalny | Ograniczony sposób działania stosowany, gdy pełna archiwizacja, migracja albo odtworzenie nie są możliwe, ale trzeba zabezpieczyć dostępne dane i decyzje. |
| Wycofanie | Pojęcie nadrzędne oznaczające kontrolowane zakończenie publikacji zasobu, eksploatacji systemu albo innego sposobu bieżącego użycia po ocenie skutków i dalszego dostępu. |
| Wycofanie systemu | Kontrolowane zakończenie eksploatacji systemu po inwentaryzacji, migracji albo archiwizacji danych, teście odtworzenia i decyzji o dalszym dostępie. |
| Wycofanie z publikacji | Kontrolowane usunięcie treści z widoku publicznego po decyzji określającej podstawę, status, miejsce zachowania i sposób dalszego dostępu. |

## Dostęp i dostępność

| Pojęcie | Definicja |
| --- | --- |
| Dostęp | Możliwość uzyskania, odczytania albo obsłużenia zasobu przez użytkownika, pracownika lub uprawnioną komórkę zgodnie z decyzją o statusie. |
| Dostęp alternatywny | Sposób przekazania informacji użytkownikowi w innej formie, gdy zasób nie spełnia wymagań dostępności cyfrowej albo nie może zostać niezwłocznie dostosowany. |
| Dostęp bieżący | Dostęp do zasobu używanego w aktualnym działaniu podmiotu, przed nadaniem statusu archiwalnego albo wycofaniem z publikacji. |
| Dostęp na wniosek | Tryb udostępnienia zasobu po zgłoszeniu potrzeby przez użytkownika, gdy zasób nie jest publicznie opublikowany, ale może podlegać udostępnieniu. |
| Dostęp ograniczony | Dostęp zawężony ze względu na dane osobowe, bezpieczeństwo informacji, tajemnicę ustawowo chronioną, status dokumentacji albo inną udokumentowaną podstawę. |
| Dostęp publiczny | Dostęp do zasobu bez indywidualnego wniosku, zwykle przez BIP, serwis WWW, repozytorium albo inny publiczny kanał podmiotu. |
| Dostępność cyfrowa | Spełnienie wymagań pozwalających osobom ze szczególnymi potrzebami korzystać z zasobu cyfrowego, w tym z treści archiwalnej. |

## Incydenty i ryzyka

| Pojęcie | Definicja |
| --- | --- |
| Incydent archiwizacji | Zdarzenie polegające na utracie, usunięciu, uszkodzeniu, niepełnej archiwizacji, utracie metadanych albo braku możliwości odczytu zasobu. |
| Incydent ochrony danych | Zdarzenie dotyczące utraty, ujawnienia, nieuprawnionego dostępu albo braku kontroli nad danymi osobowymi, które wymaga oceny z udziałem IOD. |
| Brak możliwości odczytu | Sytuacja, w której zasób istnieje, ale nie można go użyć z powodu formatu, uszkodzenia, braku oprogramowania, błędnej konwersji albo utraty zależności. |
| Ryzyko resztkowe | Ryzyko, które pozostaje po wykonaniu działań ograniczających i wymaga świadomego przyjęcia, dalszego monitorowania albo eskalacji. |
| Utrata dostępu | Sytuacja, w której znany zasób nie może zostać uzyskany przez właściwe osoby lub użytkowników z powodu uprawnień, awarii, utraty konta albo wyłączenia systemu. |
| Utrata kontekstu | Sytuacja, w której zasób istnieje, ale brakuje powiązania ze sprawą, decyzją, publikacją, właścicielem albo historią zmian. |
| Utrata metadanych | Sytuacja, w której brakuje danych opisowych potrzebnych do identyfikacji, oceny, dostępu, migracji albo archiwizacji zasobu. |

## Role i odpowiedzialność

| Pojęcie | Definicja |
| --- | --- |
| Administrator BIP | Rola odpowiedzialna za techniczne i organizacyjne wykonanie publikacji, zmian i wycofań w BIP po właściwej decyzji merytorycznej. |
| Inspektor ochrony danych | Rola oceniająca ryzyka ochrony danych osobowych oraz opiniująca dalszą publikację, ograniczenie dostępu, anonimizację i migrację danych osobowych. |
| IT | Rola wykonująca czynności techniczne, w tym eksport, kopie, zabezpieczenie integralności, migrację, test odtworzenia i wsparcie wycofania systemu. |
| Kierownictwo | Rola zatwierdzająca decyzje o istotnym skutku dla podmiotu, wyznaczająca właścicieli i przyjmująca ryzyko resztkowe, gdy wymaga tego sprawa. |
| Koordynator dostępności | Rola oceniająca dostępność cyfrową zasobu i rekomendująca naprawę, wersję dostępną albo dostęp alternatywny. |
| Redaktor | Rola wykonująca czynności publikacyjne, oznaczająca status treści, aktualizująca linki i usuwająca treść z widoku publicznego po decyzji. |
| Właściciel systemu | Osoba albo komórka odpowiedzialna za system, w którym zasób jest przechowywany, przetwarzany, publikowany, migrowany albo wycofywany. |
| Właściciel zasobu | Osoba albo komórka odpowiedzialna za znaczenie merytoryczne zasobu, aktualność, decyzję o statusie i sposób dalszego dostępu. |
| Wykonawca zewnętrzny | Podmiot wykonujący czynności umowne dotyczące systemu, eksportu, migracji, dokumentacji albo wsparcia, bez samodzielnego podejmowania decyzji merytorycznych. |

## Skróty i standardy

| Pojęcie | Definicja |
| --- | --- |
| BIP | Biuletyn Informacji Publicznej, czyli podstawowy publiczny kanał publikacji informacji wymaganych od podmiotu publicznego. |
| CMS | System zarządzania treścią używany do tworzenia, publikowania, oznaczania, archiwizacji albo wycofywania treści w serwisie. |
| EN 301 549 | Europejska norma używana do oceny wymagań dostępności cyfrowej produktów i usług TIK, w tym serwisów i dokumentów. |
| EZD | Elektroniczne zarządzanie dokumentacją, czyli system lub sposób prowadzenia spraw, dokumentów i akt w postaci elektronicznej. |
| IOD | Inspektor ochrony danych, czyli rola włączana przy ocenie publikacji, ograniczenia dostępu, retencji, migracji i incydentów dotyczących danych osobowych. |
| JRWA | Jednolity rzeczowy wykaz akt, który służy do klasyfikowania dokumentacji, ustalania kategorii archiwalnej i okresu przechowywania. |
| KRI | Krajowe Ramy Interoperacyjności, czyli wymagania organizacyjne i techniczne istotne dla systemów TIK podmiotów publicznych. |
| PDF | Format dokumentu często używany do publikacji i archiwizacji, który wymaga oceny dostępności, odczytywalności i powiązania z metadanymi. |
| RODO | Ogólne rozporządzenie o ochronie danych, które wpływa na ocenę publikacji, retencji, ograniczenia dostępu i migracji danych osobowych. |
| SDC | Nazwa repozytorium i zestawu materiałów porządkujących zalecenia dotyczące dostępności cyfrowej oraz zarządzania zasobami TIK. |
| TIK | Technologie informacyjno-komunikacyjne obejmujące systemy, usługi, dane, dokumenty i kanały cyfrowe używane przez podmiot. |
| URL | Adres zasobu w systemie lub serwisie, który pomaga ustalić miejsce publikacji, historię dostępu i potrzebę przekierowania. |
| WCAG | Wytyczne dostępności treści internetowych używane do oceny i poprawy dostępności cyfrowej stron, dokumentów i multimediów. |
| WWW | Serwis internetowy podmiotu, w którym publikowane, archiwizowane albo wycofywane są treści publiczne i informacyjne. |

## Zasady stosowania

Jeżeli pojęcie ma znaczenie prawne w przepisach albo dokumentacji wewnętrznej podmiotu, należy stosować definicję prawną albo organizacyjną. Definicje słownika nie zastępują JRWA, instrukcji kancelaryjnej, instrukcji archiwalnej ani przepisów szczególnych.
