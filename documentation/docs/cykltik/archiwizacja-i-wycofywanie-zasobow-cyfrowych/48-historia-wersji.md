# Historia wersji

## Cel

Celem historii wersji jest dokumentowanie zmian w zaleceniu oraz wskazanie, czy dokument ma charakter roboczy, wdrożeniowy albo zaktualizowany po przeglądzie.

## Zasada aktualizacji dokumentu

Historia wersji powinna być aktualizowana przy każdej zmianie merytorycznej, w szczególności przy zmianie podstaw prawnych, modeli decyzyjnych, procedur, list kontrolnych, formularzy, schematów, słownika albo źródeł. Zmiany redakcyjne można grupować, jeżeli nie wpływają na sens procedury.

Aktualizacja powinna wskazywać wersję, datę i zwięzły opis zakresu zmian. Jeżeli zmiana wpływa na sposób postępowania z zasobami cyfrowymi, należy sprawdzić powiązane dokumenty i rejestry.
